Moving Forward With Cordova Plugins

Why am I talking to you?

  • You want to build a mobile app
  • You want to reuse the code base on multiple platforms
  • You need to use some native code for specific devices
  • You need a quick start to developing native plugins
  • You want to see whats hardships are involved with Cordova/Phonegap

What this talk is not

My assumption is you've already chosen to have a hybrid application and youre wanting to know/understand how Cordova / Phonegap interacts with the native plugins.


I recommend you go see Hybrid vs Native: Choosing a Mobile Strategy by Jeff French @ 3:20

Cordova Clarification

  • Cordova is the tech that bridges Javascript to native code
  • Cordova is an Apache project (open source project)
  • Phonegap is owned by Adobe
  • Phonegap is a special fork of Cordova to be able to provide special build services in the cloud
  • Phonegap is essentially Cordova

A quick story

Two years ago, we started RaiseMore, a mobile app & platform to raise donations for charities. We already had an iOS app, but our devs were by no means native mobile developers.

What we first tried

We first used a mobile dev shop to produce iOS & Android versions / fixes / etc since we were busy with the web API. Got really expensive FAST.

Low budget, no budget, no problem

We had two devs, decided to bring mobile dev in house. No time to maintain two separate code bases. Looked at PhoneGap, Titanium, & Sencha touch. Decided to embrace Phonegap

How it turned out

Today we have an iOS app & an Android app - running on the same code base, native plugins to handle other tasks HTML cannot handle, and reuse some of the code on the mobile web version in safari & throughout our other web projects. Its AWESOME.

Where it is today

Today with Phonegap/Cordova 3.x - everything is a plugin.
